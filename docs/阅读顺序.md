# 阅读顺序

如果你明天或以后想认真看这套工具，推荐按这个顺序读。

## 第一步：先看目录

先用 `tree` 或 `find` 看整体目录。

重点先建立印象：

- 配置在哪
- 脚本在哪
- 运行时文件在哪
- 文档在哪

## 第二步：看 `README.md`

这一层先解决两个问题：

1. 它能干什么
2. 我怎么启动它

## 第三步：看 `watchdog.py`

这是最重要的代码入口。

建议重点看：

- `run_watchdog()`
- `infer_stage()`
- 状态写入
- 超时判断

## 第四步：看 `status.py`

理解它怎样把 `status.json` 变成终端 HUD。

重点看：

- `format_status_text()`
- `watch_header()`

## 第五步：看配置

- `config/stage-rules.json`
- `config/tasks.example.json`

这两份配置能帮助你理解：

- 为什么阶段会被推断成那样
- 一个任务需要哪些最小配置

## 第六步：最后再看运行态文件

建议在实际运行 watchdog 后再看：

- `runtime/status.json`
- `runtime/events.log`

这是最接近“真实工作状态”的部分。

## 建议你边看边做的事情

1. 自己准备一个会持续写日志的假任务
2. 启动 watchdog
3. 打开 `status.py --watch`
4. 对照 `events.log` 看阶段变化

这样比纯读代码更快理解它的工作方式。
