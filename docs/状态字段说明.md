# 状态字段说明

`runtime/status.json` 是 watchdog 的主状态文件。

## 核心字段

- `task_id`
  当前任务唯一标识
- `task_name`
  适合人阅读的任务名称
- `status`
  当前主状态，可能为：
  - `running`
  - `slow`
  - `stalled`
  - `stopped`
  - `completed`
  - `failed`
- `stage`
  当前阶段文字，例如：
  - `准备中`
  - `执行中`
  - `验证中`
  - `完成`
- `runtime_seconds`
  从启动到现在已经运行多久
- `idle_seconds`
  距离最后一次活跃输出过去多久
- `suggest_restart`
  是否建议人工确认后重启
- `restart_command`
  建议执行的重启命令
- `restart_count / max_restarts`
  当前已经自动恢复多少次，以及上限
- `recent_output`
  最近几行原始输出摘要。HUD 仅在 `running / slow` 时把它清洗成固定 3 行的当前任务细节，不提供历史细节回看
- `tmux_session`
  当前活跃的 `tmux session`
- `tmux_pane_id`
  当前活跃的 pane
- `tmux_pane_dead`
  pane 是否已经退出

## 设计意图

- 用 `status` 表示机器判断
- 用 `stage` 表示人类理解
- 用 `suggest_restart` 表示建议动作

这样状态展示和后续交互会比较清楚。
